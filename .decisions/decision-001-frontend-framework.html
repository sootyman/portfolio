Frontend Framework

Technical

This picks the foundation everything else is built on. For a financial dashboard, this matters a lot — you need fast page loads for charts, secure data handling, and smooth integration with AI APIs. The right pick here makes the other decisions easier; the wrong one means fighting the framework constantly. Think about how much you want to set up yourself vs. having things "just work" out of the box.

Next.js is a React framework that handles both the frontend (what users see) and the backend (API calls to databases and AI) in one project. This is ideal for a financial dashboard because sensitive API keys stay server-side and never reach the browser, and you can render data-heavy pages quickly. It's the most widely used full-stack React framework and has excellent support for streaming AI responses — important for the portfolio advisor feature.

Works well when

  • You need API keys hidden server-side
  • Fast initial page loads matter (SSR)
  • Streaming AI responses required
  • One team manages everything

Watch out for

  • More opinions baked in — less flexible
  • Slightly steeper learning curve than plain React

React with Vite gives you a pure client-side app (a Single Page Application — everything runs in the browser) with a completely separate backend you build yourself. Vite is the fastest development tool available, so the dev experience is excellent. The trade-off is you need to build and maintain a separate backend server, and you have to be careful about where your API keys live.

Works well when

  • You want total control over architecture
  • Backend is already built or separate team owns it
  • Fastest dev server experience matters

Watch out for

  • Two separate projects to manage and deploy
  • API keys need careful handling — can't be in browser
  • More setup work upfront

Vue is React's main rival — a component-based UI framework with a reputation for being approachable and having a gentle learning curve. Nuxt is its full-stack wrapper (like Next.js but for Vue). If you or your team already know Vue, this is a great choice. The ecosystem is smaller than React's but very mature, and dashboard component libraries like PrimeVue are excellent.

Works well when

  • Team already knows Vue
  • You prefer Vue's template syntax over JSX
  • PrimeVue's chart/grid components appeal

Watch out for

  • Smaller ecosystem than React — fewer AI/fintech libs
  • Less community content for financial dashboard patterns

Svelte is a compiler-based framework — instead of shipping a runtime (like React or Vue do), it compiles your components down to tiny, fast vanilla JavaScript. SvelteKit is its full-stack wrapper. The result is exceptionally fast, lightweight apps. The trade-off is the smallest ecosystem of the four options here, and fewer people know it, so finding help or hiring is harder.

Works well when

  • Performance is the top priority
  • You want minimal bundle size
  • You enjoy learning new things

Watch out for

  • Smallest ecosystem — fewer ready-made components
  • Harder to hire for / find help online
  • AI/fintech library support is limited
