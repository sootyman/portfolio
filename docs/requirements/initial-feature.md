# Basic Portfolio & Dashboard

## Overview

Two pages: a public portfolio page and a dashboard page allowing a user to enter a high level view of their portfolio and view performance with some basic analytics.

## Requirements

- Portfolio page allows the user to input the holdings in their investment portfolio - should include things like asset type and ID (ticker) for equities, quantity, purchase price, purchase date, effective date.
- Dashboard page should display the holdings in the user's portfolio in a graphical format, showing performance over a user selectable period
- Portfolio should be persisted to a data layer
- Identify a suitable data source for asset prices and historical performance
- Consider creating a local database to reduce API calls and allow user correction of prices

## Acceptance Criteria

- A user can enter a set of holdings with asset type, asset id, quantity, purchase price, purchase date, effective date 
- The dashboard displays the holdings in the user's portfolio in a graphical format, showing performance over a user selectable period
